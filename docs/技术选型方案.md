# 技术选型方案（v2：PocketBase + React）

本仓库当前只维护 **v2**（PocketBase + React + Capacitor）。本文档解释为什么选择这套架构，以及各模块的职责边界。

---

## 1. 目标与原则

### 1.1 目标

- **低运维**：后端尽量“单文件可跑”，避免传统后端的复杂环境与依赖（Python/DB/Redis 等）。
- **多端统一**：同一套前端代码同时支持 Web（PC/移动浏览器）+ Android APK（Capacitor）。
- **快速交付**：数据模型可在 PocketBase 后台快速迭代，前端直接通过 SDK CRUD。
- **可扩展**：后续需要更强后端能力时，可在 PocketBase 上方增加 BFF/网关，而不用推倒重来。

### 1.2 原则

- **v2 口径唯一**：所有文档与脚本以 `v2/` 为主线。
- **配置不写死**：后端地址、管理员凭据、AI Key 等全部通过环境变量或本机 localStorage 管理。
- **数据留痕**：关键动作写入 `audit_logs`，便于追溯与 AI 分析。

---

## 2. 后端：PocketBase（SQLite + REST + 管理后台）

### 2.1 为什么是 PocketBase

- **单文件后端**：一个可执行文件 + `pb_data/` 就能提供 API、认证、管理后台、文件存储。
- **快速建模**：集合（Collections）与字段（Schema）在后台可视化配置。
- **前端直连**：配合 PocketBase JS SDK，前端可直接调用 `pb.collection('tasks')...`。

### 2.2 我们在 PocketBase 上做什么

- 业务集合：`projects`、`tasks`、`handoffs`、`comments`、`audit_logs`、`notifications`、`ai_summaries`
- 权限规则（Rules）：实现 RBAC（`admin/manager/employee`）与字段级限制（例如员工仅能改 status/blocker）
- 数据存储：SQLite（`v2/backend/pb_data/data.db`）

---

## 3. 前端：React + TypeScript + Vite + Ant Design Mobile

### 3.1 React + TypeScript

- **组件化**适合复杂业务页面（看板、抽屉详情、时间轴）。
- TypeScript 提供更好的**类型约束与重构能力**。

### 3.2 Vite

- 开发启动快、构建快，适合频繁迭代。
- 通过 `VITE_*` 环境变量注入运行配置（例如 `VITE_PB_URL`）。

### 3.3 Ant Design Mobile

- 移动端组件成熟，适合“触控优先”的产品形态。
- 同时在 PC 浏览器也能良好表现（配合响应式布局）。

---

## 4. 数据与状态：TanStack Query + Zustand

- **TanStack Query**：管理服务端数据（projects/tasks/users 等），处理缓存、刷新、重试。
- **Zustand**：管理 UI 状态（筛选条件、拖拽中的任务 ID、草稿等）。

---

## 5. 移动端：Capacitor

### 5.1 为什么是 Capacitor

- 让 Web 前端直接打包成 Android APK，成本低、迭代快。
- 对接原生能力（例如通知、文件、相机）时可按需扩展插件。

### 5.2 重要注意事项

- App 内 WebView 通常运行在 `localhost`，所以 **PocketBase 地址必须通过 `VITE_PB_URL` 或 `localStorage.getItem('pb_url')` 指定**（详见 `v2/docs/App打包指南.md` 与 `v2/frontend/docs/APK_BUILD_GUIDE.md`）。

---

## 6. AI：SiliconFlow（DeepSeek）

- **用途**：管理端生成风险简报、问答辅助分析。
- **Key 管理**：
  - 当前：由经理/管理员在前端配置并存入本机 `localStorage('sf_api_key')`
  - 生产建议：增加 BFF（服务端代理），避免 Key 下发到前端

---

## 7. 部署形态

- **后端**：PocketBase 常驻进程（建议内网/本机监听 8090，通过 Nginx `/pb` 反代）
- **前端**：Vite 构建产物 `dist/` 部署到 Nginx 静态站点（SPA 路由回退到 `index.html`）

部署细节见：

- `v2/docs/宝塔部署操作手册.md`
