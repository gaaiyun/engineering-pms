## 一、产品概述

### 1.1 产品定位

工程结算管理系统（EngineeringPMS）是一款面向工程建设行业的**移动优先**项目管理工具，支持 Web 端和 Android 端。系统围绕「项目→任务→人员→审计」四层模型，为项目经理和一线员工提供任务分配、进度追踪、卡点上报、变更审计、消息通知等核心能力。

通过"三步法"方法论（标准化拆解、可视化追踪、流程化配合）解决多项目并行场景下的信息断层与协作内耗问题。

### 1.2 核心价值

- **角色分层**：经理掌控全局（创建/编辑/删除/归档项目、批量分配任务），员工聚焦执行（只看自己参与的项目和任务）。
- **变更留痕**：所有操作自动写入审计日志，经理可在变更审计中心逐条复核。
- **实时同步**：基于 PocketBase SSE 的实时数据订阅，操作即刷新，多端同步。
- **移动适配**：100dvh + safe-area + 横屏旋转，适配刘海屏和全面屏。
- **AI 赋能**：智能风险诊断与问答助手（SiliconFlow / DeepSeek）。

### 1.3 目标用户

| 角色 | 核心诉求 | 使用频率 |
|---|---|---|
| 老板/决策层 | 全局掌控、风险预警、AI 简报 | 每天 1-2 次 |
| 项目经理 | 流程管控、团队协调、审核交接 | 每天 5-10 次 |
| 执行层（员工） | 明确任务、高效执行、卡点上报 | 每天 10+ 次 |

## 二、角色与权限体系

### 2.1 角色定义

| 角色 | 对应 role 字段 | 说明 |
|---|---|---|
| 管理员/经理 | `admin` 或 `manager` | 拥有所有权限，包括创建/编辑/删除/归档项目、管理成员、分配任务、审核变更 |
| 普通员工 | `employee` | 仅能查看自己参与的项目和被分配的任务，不能编辑项目或指派他人 |

### 2.2 权限矩阵

| 操作 | 管理员/经理 | 普通员工 |
|---|---|---|
| 查看所有项目 | 是 | 否（仅参与的项目） |
| 创建项目 | 是 | 否 |
| 编辑项目 | 是 | 否 |
| 删除项目 | 是 | 否 |
| 归档/取消归档项目 | 是 | 否 |
| 管理项目成员 | 是 | 否 |
| 创建/编辑/删除任务 | 是 | 否 |
| 批量编辑任务 | 是 | 否 |
| 查看所有任务 | 是 | 否（仅被分配的） |
| 上报卡点 | 是 | 是（仅自己的任务） |
| 标记任务完成 | 是 | 是（仅自己的任务） |
| 进入管理控制台 | 是 | 否 |
| 进入审核中心 | 是 | 否 |
| 查看消息通知 | 是 | 是 |

### 2.3 路由与界面

- **经理/管理员**：登录或冷启动后进入 `/admin`（五 Tab 界面：概览、时间轴、项目、AI、我的）。
- **普通员工**：登录或冷启动后进入 `/app`（三 Tab 界面：工作进展、管理入口隐藏、我的）。
- **DefaultRedirect**：根据 `pb.authStore.model.role` 自动判断跳转目标，`path="/"` 和 `path="*"` 均使用此组件。

## 三、功能模块详细说明

### 3.1 项目管理

#### 3.1.1 创建项目

- **入口**：
  - **MyProjects 页面**：右上角"+ 新建项目"按钮（经理可见）
  - **Admin 控制台 - 项目 Tab**：右上角"+ 新建"按钮（经理可见）
  - **Home 页面（员工端）**：右下角浮动"+"按钮（经理可见）
- **组件**：`BatchProjectCreator`，支持填写项目名称、描述、截止日期、选择成员，并可**同时批量添加任务**（三列布局：任务名 | 执行人 | 截止时间）
- **数据流**：`useCreateProject` → PocketBase `projects` 集合 → 自动创建审计日志 + 通知项目成员
- **特点**：
  - 一步完成项目创建和任务分配，无需二次操作
  - 任务可选，支持创建空项目后再添加任务
  - 所有创建项目的入口统一使用同一组件，用户体验一致

#### 3.1.2 Excel 快速导入（高级功能）

- **入口**：MyProjects 页面右上角"📊 Excel 导入"按钮（经理可见）
- **组件**：`ThreeColumnImport`，支持从 Excel 粘贴三列格式批量导入
- **格式**：每行一个任务，用 Tab 或多个空格分隔
  - 第 1 列：任务名称（必填）
  - 第 2 列：负责人（姓名/用户名/邮箱，可选）
  - 第 3 列：截止日期（可选）
- **智能解析**：
  - 日期格式：2026-02-20 | 02-20 | 2/20 | +7天 | +2w周 | 2月20日
  - 人员匹配：自动匹配系统中的用户
  - 实时验证：显示有效任务数、错误和警告
- **适用场景**：需要从 Excel 导入大量任务时使用

#### 3.1.3 编辑项目

- **入口**：MyProjects 项目卡片上的编辑按钮（仅经理可见）。
- **支持修改**：项目名称、描述、状态、截止日期。
- **数据流**：`useUpdateProject` → 审计日志 → 通知。

#### 3.1.4 管理项目成员

- **入口**：MyProjects 项目卡片上的成员按钮。
- **交互**：弹出成员弹窗，搜索用户、勾选/取消勾选、保存。
- **数据流**：`useUpdateProjectMembers` → 审计日志（action_type: `update_members`） → 通知。

#### 3.1.5 删除项目

- **入口**：MyProjects 项目卡片上的删除按钮（仅经理可见）。
- **行为**：二次确认弹窗 → 级联删除项目下所有任务 → 创建审计日志 → 通知所有成员。
- **数据流**：`useDeleteProject`。

#### 3.1.6 归档与取消归档

- **入口**：MyProjects 项目卡片上的归档/取消归档按钮。
- **规则**：
  - 已归档项目**不计入**概况统计（KPI 卡片、饼图、逾期率等）。
  - 已归档项目仍在项目列表中可见（通过「已归档」Tab 筛选）。
- **数据流**：`useArchiveProject` → 审计日志 → 通知。

#### 3.1.7 项目列表分类筛选

四个 Tab：

| Tab | 筛选条件 |
|---|---|
| 全部 | 所有项目 |
| 进行中 | `status === 'active'` 且不在卡顿列表中 |
| 卡顿 | `status === 'active'` 且有任务阻塞超过 7 天 |
| 已归档 | `status === 'archived'` |

### 3.2 任务管理

#### 3.2.1 批量任务编辑

- **入口**：
  - **ProjectKanban 页面**：顶部"批量编辑任务"按钮（仅经理可见）
  - **ProjectTimeline 页面**：右下角紫色圆形按钮（仅经理可见）
- **组件**：`BatchTaskEditor`，三列布局（任务名称 | 执行人员 | 截止时间）。
- **支持**：增行、删行、一次性保存。
- **数据流**：`useBatchSaveTasks` → 批量创建/更新任务 → 审计日志 → 通知。
- **数据流**：`useBatchSaveTasks` → 审计日志 → 通知。

#### 3.2.2 卡点上报

- **简化**：只需填写「卡点原因」和「预计解决日期」，无需选择类型（`reason_type` 硬编码为 `'other'`）。
- **两个入口统一**：TaskDetail 页面和 TaskDetailDrawer（看板版）逻辑一致。
- **取消卡点**：经理可选择「标记已完成」或「恢复为进行中」。

#### 3.2.3 任务状态流转

```
pending → in_progress → completed
                ↓
            blocked (卡点)
                ↓
        completed 或 in_progress (解除卡点)
```

### 3.3 变更审计中心（ReviewCenter）

#### 3.3.1 功能概述

所有项目/任务变更自动记录到 `audit_logs` 集合。变更审计中心提供经理对变更的复核界面，变更自动生效，复核仅为留痕。

#### 3.3.2 覆盖的变更类型（15 种）

| action_type | 标签 |
|---|---|
| `update_task` | 任务修改 |
| `create_task` | 创建任务 |
| `mark_complete` | 任务完成 |
| `mark_blocked` | 卡点上报 |
| `unblock_task` | 卡点解除 |
| `batch_edit_tasks` | 批量编辑 |
| `update_members` | 成员变更 |
| `archive_project` | 项目归档 |
| `delete_task` | 删除任务 |
| `delete_project` | 删除项目 |
| `create_project` | 创建项目 |
| `update_project` | 项目修改 |
| `unarchive_project` | 取消归档 |
| `approve_handoff` | 交接通过 |
| `reject_handoff` | 交接驳回 |

#### 3.3.3 界面功能

- **搜索**：按关键字搜索变更记录。
- **筛选**：按操作类型筛选（漏斗图标展开筛选器）。
- **分类 Tab**：全部 / 待复核 / 已阅读 / 已通过 / 交接审核。
- **操作按钮**：「已阅读」和「通过」（仅待复核状态显示）。
- **卡片信息**：项目名、操作人、任务名、时间、变更详情。
- **实时同步**：SSE 订阅 `audit_logs` 和 `handoffs` 集合。

#### 3.3.4 入口

- AdminDashboard 概览：快捷操作「任务审核」、待审核卡片「去处理」。
- AdminDashboard 我的：「审核中心」。
- ManagerDashboard：「进入审核中心」按钮、异常任务列表。

### 3.4 消息通知

#### 3.4.1 触发场景

以下操作会自动通知项目内所有成员（`project.members` + `project.manager`，排除操作者本人）：

1. 创建任务
2. 修改任务（名称、人员、时间、状态等）
3. 标记任务完成
4. 删除任务
5. 上报卡点
6. 解除卡点
7. 批量编辑任务
8. 修改项目信息
9. 变更项目成员
10. 归档/取消归档项目
11. 删除项目

#### 3.4.2 实时刷新

- `Notifications.tsx` 组件内直接 SSE 订阅 `notifications` 集合。
- `main.tsx` 全局 `subscribeToChanges` 订阅 6 个集合（tasks/projects/handoffs/notifications/audit_logs/comments），变更时自动 invalidate 对应 query。
- 登录状态变化时自动重新订阅。

### 3.5 管理控制台（AdminDashboard / 五 Tab）

#### 3.5.1 概览 Tab

- **KPI 卡片**（4个，可点击跳转）：

- **统计口径**：KPI 和图表均基于 `activeTasks`（排除归档项目的任务），与经理工作台数据一致。

#### 3.5.2 时间轴 Tab

AI 控制台，支持项目数据分析、智能建议。

#### 3.5.5 我的 Tab

个人资料、我的项目、我的任务、审核中心入口、设置。

### 3.6 员工首页（Home）

- **工作进展 Tab**：
  - 员工视角：显示「我的当前任务」（进行中和待办，最多 5 条）和「未读消息」（最多 5 条），可点击跳转。
  - 经理视角：显示 `Tasks` 组件（项目与任务进度）。

### 3.7 时间轴（ProjectTimeline）

- **PC/平板**：甘特图式时间轴布局。

## 四、数据模型

### 4.1 集合结构

| 集合名 | 说明 | 核心字段 |
|---|---|---|
| `users` | 用户 | username, name, email, role, department, avatar |
| `projects` | 项目 | name, project_number, description, status, manager, members, deadline |
| `tasks` | 任务 | stage_name, project, assignees, status, deadline, priority, blocker, score, approved |
| `notifications` | 通知 | user, type, title, content, is_read, link_type, link_id |
| `audit_logs` | 审计日志 | project, task, operator, action_type, before_data, after_data, note, review_status |
| `handoffs` | 交接记录 | from_task, submitter, proposed_title, proposed_assignees, proposed_due_date, status |
| `comments` | 评论 | task, user, content |

### 4.2 数据过滤逻辑

- **员工**：`useProjects` 按 `members ~ userId` 过滤，回退按 `assignees` 反查；`useTasks` 按 `assignees ~ userId` 过滤。
- **经理**：查看所有项目和任务。
- **归档排除**：`activeTasks = tasks.filter(t => !archivedProjectIds.has(t.project))`，用于 KPI 统计。

## 五、部署指南

### 5.1 环境要求

- Node.js 18+
- PocketBase 0.22+
- Android Studio（用于 APK 打包）
- 服务器：任意 Linux/Windows（推荐宝塔面板）

### 5.2 后端部署

```bash
cd backend
./pocketbase serve --http 0.0.0.0:8090
```

### 5.3 前端构建

```bash
cd frontend
npm install
VITE_PB_URL=http://your-server:8090 npm run build
```

### 5.4 Web 部署

将 `frontend/dist` 目录部署到 Nginx/Apache，配置 SPA fallback：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

### 5.5 Android APK 打包

```bash
cd frontend
npx cap sync
cd android
./gradlew assembleDebug
```

APK 输出：`android/app/build/outputs/apk/debug/app-debug.apk`

### 5.6 自动备份

**Linux（crontab）：**

```bash
0 */12 * * * cd /path/to/project && node scripts/auto_backup.mjs >> /var/log/pb_backup.log 2>&1
```

**Windows（计划任务）：**

```powershell
schtasks /create /tn "PB_Backup" /tr "powershell -File C:\path\to\backend\backup.ps1" /sc hourly /mo 12
```

策略：每 12 小时自动备份，保留 60 天，sqlite3 安全备份（含 WAL）。

## 六、使用场景与流程

### 6.1 经理创建新项目并分配任务

1. 登录 → 自动进入五 Tab 界面（/admin）。
2. 切到「项目」Tab → 点击右下角 `+`。
3. 填写项目基本信息（名称、编号、截止日期）。
4. 勾选项目成员。
5. 在下方批量添加任务（三列式：任务名 | 执行人 | 截止时间）。
6. 点击「创建项目」→ 项目和任务一次性创建完成。
7. 所有被分配的成员自动收到通知。

### 6.2 员工执行任务并上报卡点

1. 登录 → 进入三 Tab 界面（/app）。
2. 首页显示「我的当前任务」和「未读消息」。
3. 点击任务卡片 → 进入任务详情。
4. 若遇阻塞 → 点击「上报卡点」→ 填写原因和预计解决日期。
5. 卡点信息自动通知项目全员和经理。

### 6.3 经理审核变更记录

1. 在概览页点击「进入审核中心」或待审核卡片的「去处理」。
2. 进入变更审计中心，查看所有变更记录。
3. 可按类型筛选（任务修改/成员变更/卡点上报等）。
4. 逐条点击「已阅读」或「通过」标记复核状态。

### 6.4 项目归档

1. 在项目列表找到已完成的项目。
2. 点击归档按钮 → 确认。
3. 归档后：
   - 项目从概览 KPI 统计中排除。
   - 项目仍可在「已归档」Tab 中查看。
   - 可随时取消归档恢复为活跃状态。

| `/api/collections/handoffs/records` | POST | 创建交接提案 |
| `/api/collections/notifications/records` | GET | 通知列表 |

### 8.3 前端 API Hooks

**查询 Hooks**：`useProjects`, `useProject(id)`, `useTasks(projectId?)`, `useTask(id)`, `useMyTasks`, `useUsers`, `useCurrentUser`, `useHandoffs`, `usePendingHandoffs`, `useNotifications`, `useUnreadNotificationCount`, `useAuditLogs`, `useComments`, `useAISummaries`

**变更 Hooks**：`useUpdateTask`, `useCreateTask`, `useDeleteTask`, `useCreateHandoff`, `useApproveHandoff`, `useRejectHandoff`, `useMarkTaskComplete`, `useMarkTaskBlocked`, `useUnblockTask`, `useCreateComment`, `useMarkNotificationRead`, `useBatchSaveTasks`, `useCreateProject`, `useDeleteProject`, `useUpdateProject`, `useUpdateProjectMembers`, `useArchiveProject`

### 8.4 实时订阅

`pocketbase.ts` 中 `subscribeToChanges()` 订阅 6 个集合：tasks, projects, notifications, handoffs, audit_logs, comments。变动时自动 `invalidateQueries` 刷新前端缓存。

## 十三、已知限制与后续规划

1. **审核中心搜索范围**：目前搜索仅覆盖 `note` 和 `action_type` 字段，不包含项目名/任务名的全文搜索。

3. **E2E 测试**：仅有 smoke 测试，未覆盖冷/热启动和完整用户流程。
4. **消息推送**：目前仅站内通知，后续可集成 FCM/APNs。
5. **AI Key 安全**：当前存储在 localStorage，建议迁移到服务端代理。
