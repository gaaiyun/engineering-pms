## 一、技术栈概览

| 层级 | 技术 | 版本 | 用途 |
|------|------|------|------|
| 前端框架 | React | 19.x | UI 构建 |
| 类型系统 | TypeScript | 5.9+ | 类型安全 |
| 构建工具 | Vite | 6.0+ | 开发&打包 |
| UI 组件库 | Ant Design Mobile | 5.42+ | 移动端组件 |
| 状态管理 | TanStack Query + Zustand | 5.x | 服务端/客户端状态 |
| 动画库 | Framer Motion | 12.x | 微动效 |
| 图表 | ECharts + Recharts | 5.x | 数据可视化 |
| 拖拽 | @dnd-kit | - | 看板拖拽 |
| 后端 | PocketBase | 0.26.3 | BaaS (含 Realtime) |
| AI 服务 | SiliconFlow (DeepSeek) | - | 智能分析 |
| 移动打包 | Capacitor | 5.x | Android APK |

### 选型理由

| 技术 | 选择原因 |
|------|---------|
| PocketBase | 单文件后端（exe + `pb_data` 即完整服务），前端 JS SDK 直连 CRUD，后台可视化建模快速迭代 |
| React + TypeScript | 组件化适合复杂业务页面（看板、抽屉、时间轴），TS 提供类型约束与重构安全 |
| Vite | 启动快构建快，通过 `VITE_*` 环境变量注入运行配置 |
| Ant Design Mobile | 移动端组件成熟（触控优先），PC 浏览器同样良好表现 |
| TanStack Query + Zustand | Query 管理服务端数据缓存/刷新/重试；Zustand 管理纯 UI 状态（筛选、草稿等） |
| Capacitor | Web 前端直接打包 Android APK，成本低迭代快，可按需扩展原生插件 |

### 设计原则

- **低运维**：后端单文件可跑，避免传统后端复杂环境依赖
- **多端统一**：同一套前端代码支持 Web（PC/移动浏览器）+ Android APK
- **配置不写死**：后端地址、AI Key 等通过环境变量或 localStorage 管理
- **数据留痕**：关键动作写入 `audit_logs`，便于追溯与 AI 分析

## 七、状态机与关键流程

### 7.1 任务状态流转

```
                  ┌──────────────────────────┐
                  │        pending           │
                  └────────────┬─────────────┘
                     员工开始任务│(写入 start_date)
                               ▼
                  ┌──────────────────────────┐
              ┌───│      in_progress         │───┐
   员工上报卡点│   └──────────────────────────┘   │员工完成任务
   (写入blocker)│                                 │(写入completed_at)
              ▼                                   ▼
 ┌────────────────┐                 ┌──────────────────────────┐
 │    blocked      │                 │      completed           │
 └────────────────┘                 └──────────────────────────┘
        │ 经理解除卡点                          │
        └──→ in_progress                       │ 创建 handoff (status=pending)
                                               ▼
                                    ┌──────────────────────────┐
                                    │  经理审核交接提案          │
                                    │  通过 → approved + 创建   │
                                    │         下一条 task       │
                                    │  驳回 → rejected +        │
                                    │         review_note       │
                                    └──────────────────────────┘
```

### 7.2 角色操作边界

| 角色 | 允许的状态变更 | 关键约束 |
|------|--------------|---------|
| 员工 (employee) | `pending→in_progress`, `in_progress→blocked`, `in_progress→completed` | 仅能改 status/blocker，不能编辑任务其他字段 |
| 经理 (manager) | 所有状态变更 + 审核交接 | 通过交接时自动创建下一条任务 |

### 7.3 前端关键交互

- **Kanban 拖拽**：改变 `tasks.status`，列内排序更新 `sequence`
- **Timeline 渲染**：基于 `start_date` / `deadline` / `completed_at` 绘制甘特图
- **通知推送**：订阅 `notifications` 集合（Realtime SSE），轮询兜底

### 配置要点

- **PB 地址（前端）**：`VITE_PB_URL` > `localStorage.pb_url` > 运行时推导（HTTP 用 `host:8090`，HTTPS 用同域 `/pb`）

- **数据存储**：SQLite（`backend/pb_data/data.db`）
- **权限规则**：通过 `database_rebuild.mjs` 配置 `managerOnlyRules`

## 十四、测试规范

### 14.1 测试技术栈

| 工具 | 用途 |
|------|------|
| Vitest | 测试运行器 + 断言库 |
| @testing-library/react | 组件渲染与交互测试 |
| @testing-library/jest-dom | DOM 断言扩展 |
| @testing-library/user-event | 用户交互模拟 |
| happy-dom | 浏览器环境模拟 |
| @vitest/coverage-v8 | 覆盖率收集 |

命令均在 `frontend/` 目录下执行：`npm test`（全部）、`npm run test:watch`（监听）、`npm run test:coverage`（覆盖率）。

### 14.2 目录与命名约定

- 测试文件与源码**同目录**放置，命名：`<源文件名>.test.ts(x)`
- 全局 setup 文件：`src/test/setup.ts`
- 覆盖率配置：`vitest.config.ts` 的 `test.coverage` 字段

### 14.3 测试分层

| 层级 | 目标 | 方式 |
|------|------|------|
| 单元测试 | `lib/` 下纯函数（task-parser、queryKeys、isManager 等） | 直接调用，不依赖 DOM |
| 组件测试 | UI 组件渲染与交互 | `render` + `screen`，只测用户可见行为 |
| 集成测试 | TanStack Query hooks | `renderHook` + `QueryClientProvider`，mock `pb.collection()` |

### 14.4 用例书写规范

- 一条用例只验证一个行为
- 描述用中文，格式：`it('当 X 时，应 Y')`
- 每类输入至少覆盖：正常值、边界值、非法值
- 不测实现细节（不断言内部 state 变量名）

### 14.5 Mock 规范

- **PocketBase**：统一 mock `lib/pocketbase` 模块（`pb.authStore` + `pb.collection()`）
- **路由**：使用 `MemoryRouter` 控制初始路径
- **第三方库**：`@dnd-kit/sortable` mock `useSortable`；`@capacitor/app` mock `addListener`

### 14.6 覆盖率目标

| 范围 | 语句/分支目标 |
|------|--------------|
| `lib/task-parser.ts` | ≥ 90% |
| 新增代码 | ≥ 80% |
| 旧代码 | 逐步提升 |

新增纯函数/工具**必须**附带单元测试；新增组件**建议**附带渲染测试；PR 中测试不通过不允许合并。
