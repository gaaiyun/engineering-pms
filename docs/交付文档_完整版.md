## 1. 产品概述

### 1.1 产品定位

本产品是一款专为工程管理公司打造的**全流程协同管理平台**。通过"三步法"（标准化拆解、可视化追踪、流程化配合）解决多项目并行下的信息断层与协作内耗问题。

### 1.2 目标用户

| 角色 | 核心诉求 | 主要功能 |
|------|----------|----------|
| **老板/决策层** | 全局掌控、风险预警 | AI 简报、风险看板、异常任务聚焦 |
| **项目经理** | 流程管控、团队协调 | 项目看板、审核中心、任务编排、AI 问答 |
| **执行层（资料员）** | 明确任务、高效执行 | 我的任务、卡点上报、交接提案 |

### 1.3 核心价值

1. **标准化**：将复杂工程拆解为标准化任务节点
2. **可视化**：看板+甘特图，一眼看清进度
3. **闭环化**：强制"完成即交接"，杜绝推诿

```typescript
// src/pages/ProjectTimeline.tsx

// 计算任务条位置
const calculateTaskPosition = (task: Task, timelineStart: Date, dayWidth: number) => {
  const startDate = new Date(task.start_date || task.created)
  const endDate = new Date(task.deadline || task.created)
  
  const startOffset = Math.floor(
    (startDate.getTime() - timelineStart.getTime()) / (1000 * 60 * 60 * 24)
  )
  const duration = Math.ceil(
    (endDate.getTime() - startDate.getTime()) / (1000 * 60 * 60 * 24)
  ) + 1
  
  return {
    left: startOffset * dayWidth,
    width: duration * dayWidth,
  }
}

// 缩放处理
const handleZoom = (delta: number) => {
  setZoom(prev => Math.min(200, Math.max(50, prev + delta)))
}
```

### 2.4 交接闭环

**流程设计**：
```
员工完成任务 → 填写交接提案(Handoff) → 经理审核
                                          │
            ┌─────────────────────────────┘
            │
     ┌──────┴──────┐
     │             │
   通过          驳回
     │             │
     ▼             ▼
自动创建新任务   退回修改
通知新负责人
```

**业务规则**：
- 员工不能直接创建任务，必须通过交接提案
- 经理审核通过后系统自动创建下一步任务
- 所有操作记录到 `audit_logs`

**代码实现**（`api.ts`）：
```typescript
// 审核通过交接
export function useApproveHandoff() {
  return useMutation({
    mutationFn: async ({ handoffId, reviewNote }) => {
      const handoff = await pb.collection('handoffs').getOne(handoffId)
      
      // 1. 创建新任务
      const newTask = await pb.collection('tasks').create({
        project: handoff.project,
        stage_name: handoff.proposed_title,
        assignees: handoff.proposed_assignees,
        deadline: handoff.proposed_due_date,
        status: 'pending',
      })
      
      // 2. 更新交接状态
      await pb.collection('handoffs').update(handoffId, {
        status: 'approved',
        reviewer: pb.authStore.model.id,
        review_note: reviewNote,
        approved_task: newTask.id,
      })
      
      // 3. 发送通知给新负责人
      for (const userId of handoff.proposed_assignees) {
        await pb.collection('notifications').create({
          user: userId,
          type: 'task_assigned',
          title: '新任务分配',
          content: `您被分配了新任务：${handoff.proposed_title}`,
        })
      }
      
      // 4. 记录审计日志
      await pb.collection('audit_logs').create({
        action_type: 'approve_handoff',
        operator: pb.authStore.model.id,
        note: `审核通过：${handoff.proposed_title}`,
      })
    }
  })
}
```

```typescript
// src/lib/ai-service.ts

export async function generateAIReport(projectData: ProjectData): Promise<AIReport> {
  const apiKey = localStorage.getItem('sf_api_key')
  const model = localStorage.getItem('ai_model') || 'deepseek-ai/DeepSeek-V3'
  
  const systemPrompt = `你是一位专业的工程项目管理顾问。请基于以下项目数据，生成一份简洁的风险诊断报告。
  
输出格式（JSON）：
{
  "overall_assessment": "整体评估（1-2句话）",
  "progress_summary": "进度摘要",
  "blocked_analysis": "卡点分析（如有）",
  "risk_warnings": ["风险预警1", "风险预警2"],
  "recommendations": ["建议1", "建议2"],
  "risk_level": "low|medium|high"
}`

  const response = await fetch('https://api.siliconflow.cn/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${apiKey}`,
    },
    body: JSON.stringify({
      model,
      messages: [
        { role: 'system', content: systemPrompt },
        { role: 'user', content: JSON.stringify(projectData) },
      ],
      response_format: { type: 'json_object' },
    }),
  })
  
  const data = await response.json()
  return JSON.parse(data.choices[0].message.content)
}
```

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    root /www/wwwroot/app;
    index index.html;
    
    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # PocketBase 反代（推荐）
    location /pb/ {
        proxy_pass http://127.0.0.1:8090/;
        proxy_set_header Host $host;
```

#### 使用示例

```typescript
// 页面组件中使用 Hooks
import { useTasks, useUpdateTask, useCreateHandoff } from '@/lib/api'

function TaskList({ projectId }: { projectId: string }) {
  // 查询任务列表
  const { data: tasks, isLoading, error } = useTasks(projectId)
  
  // 更新任务状态
  const updateTask = useUpdateTask()
  
  // 创建交接提案
  const createHandoff = useCreateHandoff()
  
  const handleStatusChange = (taskId: string, newStatus: TaskStatus) => {
    updateTask.mutate({
      id: taskId,
      data: { status: newStatus }
    }, {
      onSuccess: () => Toast.show({ content: '状态已更新' }),
      onError: (err) => Toast.show({ content: '更新失败' })
    })
  }
  
  const handleCompleteWithHandoff = async (task: Task) => {
    // 先更新状态为完成
    await updateTask.mutateAsync({
      id: task.id,
      data: { status: 'completed' }
    })
    
    // 再创建交接提案
    await createHandoff.mutateAsync({
      project: task.project,
      from_task: task.id,
      proposed_title: '下一步任务',
      status: 'pending',
      submitter: pb.authStore.model?.id
    })
  }
  
  if (isLoading) return <Skeleton />
  if (error) return <ErrorState error={error} />
  
  return (
    <List>
      {tasks?.map(task => (
        <TaskCard 
          key={task.id} 
          task={task}
          onStatusChange={handleStatusChange}
          onComplete={handleCompleteWithHandoff}
        />
      ))}
    </List>
  )
}
```

- **建议**：迁移到服务端代理

### 10.2 数据备份与恢复

#### 10.2.1 备份策略

| 备份类型 | 频率 | 保留时间 | 说明 |
|----------|------|----------|------|
| 每日备份 | 每天凌晨 | 7 天 | 自动备份 |
| 周备份 | 每周日 | 4 周 | 自动备份 |
| 月备份 | 每月 1 日 | 12 个月 | 自动备份 |
| 手动备份 | 按需 | 永久 | 重大变更前 |

#### 10.2.2 备份脚本

```bash
#!/bin/bash
# backup_pocketbase.sh

BACKUP_DIR="/backup/pocketbase"
DATE=$(date +%Y%m%d_%H%M%S)
PB_DATA="/www/server/pocketbase/pb_data"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
cp $PB_DATA/data.db $BACKUP_DIR/data_$DATE.db

# 压缩备份
gzip $BACKUP_DIR/data_$DATE.db

# 清理 7 天前的备份
find $BACKUP_DIR -name "data_*.db.gz" -mtime +7 -delete

echo "Backup completed: data_$DATE.db.gz"
```

#### 10.2.3 数据恢复

```bash
# 1. 停止 PocketBase
pkill -f pocketbase

# 2. 备份当前数据（以防万一）
cp /www/server/pocketbase/pb_data/data.db /www/server/pocketbase/pb_data/data.db.bak

# 3. 恢复备份
gunzip /backup/pocketbase/data_20260204.db.gz
cp /backup/pocketbase/data_20260204.db /www/server/pocketbase/pb_data/data.db

# 4. 重启 PocketBase
cd /www/server/pocketbase
nohup ./pocketbase serve --http="0.0.0.0:8090" > pb.log 2>&1 &
```
