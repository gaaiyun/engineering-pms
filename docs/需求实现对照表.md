| 编辑项目 | 已完成 | `api.ts` useUpdateProject |
| 删除项目 | 已完成 | `api.ts` useDeleteProject（级联删除所有任务） |
| 归档/取消归档 | 已完成 | `api.ts` useArchiveProject |

| 计划变更全权经理 | 已完成 | 所有编辑操作通过 `isManagerRole()` 限制 |

### 1.2 普通员工权限

| 需求 | 状态 | 实现位置 |
|---|---|---|
| 仅看自己参与的项目 | 已完成 | `api.ts` useProjects 按 `members~userId` 过滤 |
| 不能编辑任务 | 已完成 | UI 按钮通过 `isManager()` 隐藏 |
| 不能修改项目信息 | 已完成 | 编辑/删除按钮仅经理可见 |
| 不能指派下一位执行人 | 已完成 | 交接提案仅经理可审核，员工仅可提交 |
| 不能拖拽看板卡片 | 已完成 | `KanbanBoard.tsx` sensors 对员工设为 Infinity |
| 可上报卡点 | 已完成 | `TaskDetailDrawer.tsx` 所有角色可报卡点 |
| 可标记自己的任务完成 | 已完成 | `TaskDetail.tsx` 和 `TaskDetailDrawer.tsx` 完成按钮 |

> **老板原话理解修正**：老板说"删除和编辑的权力只有管理员有（经理是管理员）"。在我们系统中 `admin` 和 `manager` 角色权限等同，都拥有管理权。`employee` 角色无编辑权。这与老板意图一致。

### 与老板原始需求的差异说明

1. **"删除项目目前未实现"** → 实际已实现（`useDeleteProject` 级联删除任务）
2. **"消息不会实时刷新"** → 已通过 PocketBase SSE + React Query 实现全链路实时刷新
3. **"横屏未实现"** → ProjectTimeline 已支持横屏检测和 CSS 适配
4. **"卡点上报要填类型"** → 已简化为只填原因（`reason_type` 硬编码为 `'other'`）
5. **"自动备份未完成"** → 脚本已编写，只需在服务器上配置定时任务
