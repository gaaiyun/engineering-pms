## 工程结算 App 快速启动（命令行版）

本指南帮你**下次一条命令就能把系统跑起来**，分为：

- **本地开发环境启动**
- **服务器（宝塔环境）重启 / 重新初始化 PocketBase**

---

## 一、本地开发环境启动

> 前提：已经在 `v2/frontend` 和 `v2/scripts` 各执行过一次 `npm install`。

### 1. 启动本地 PocketBase 后端

在 Windows 终端 / PowerShell 中：

```bash
cd "G:\项目管理软件\v2\backend"
.\pocketbase.exe serve --http="0.0.0.0:8090"
```

- 这是**本地调试后端**，端口 `8090` 与线上保持一致。
- 终端不要关，关了后端就停了。

> 小贴士：你也可以直接双击同目录下的 `启动后端.bat`，效果等同。

### 2. 启动前端（Vite + React）

新开一个终端窗口：

```bash
cd "G:\项目管理软件\v2\frontend"
npm run dev
```

浏览器访问：

- `http://localhost:5173`

默认账号（由初始化脚本创建）：

- （示例）老板/管理员：`admin_boss / 12345678`
- （示例）经理：`zhang_manager / 12345678`、`wang_manager / 12345678`
- （示例）员工：`chen_doc / 12345678`、`li_audit / 12345678`

> 若你没有运行初始化脚本，账号以你在 PocketBase 管理后台创建的为准。

## 二、服务器端（宝塔）快速重启 PocketBase

当你发现手机/前端打不开后台 `http://<YOUR_PB_HOST>:8090/_/` 时，一般是 PocketBase 没在跑。

### 1. SSH 登陆服务器

通过宝塔终端或 SSH 工具连接到服务器，然后执行：

```bash
cd /www/server/pocketbase
nohup ./pocketbase serve --http="0.0.0.0:8090" > pb.log 2>&1 &
```

含义：

- `nohup`：后台运行，不随终端关闭而退出
- 输出重定向到 `pb.log`，方便排查问题

检查是否正常：

- 浏览器打开 `http://<YOUR_PB_HOST>:8090/_/` 能看到 PocketBase 后台登录页面即可。

---

## 三、服务器端一键同步数据库结构与测试数据

当你**修改了后端结构 / 字段**，或者想**恢复一份干净的示例数据**时，可直接在本地执行初始化脚本。

### 1.（推荐）一键重建演示数据（⚠️ 会清空业务数据）

在你电脑的终端中：

```bash
cd "G:\项目管理软件\v2\scripts"
node database_rebuild.mjs
```

⚠️ 注意：`database_rebuild.mjs` 会清空并重建以下集合的数据（用于演示/联调）：

- `tasks` / `projects` / `handoffs` / `comments` / `audit_logs` / `ai_summaries` / `notifications`

请不要在已有真实数据的生产库上直接执行（除非你确认要清空并重建）。

### 2.（非清空）修复/补齐 AI 相关集合（推荐用于线上）

如果你只想确保 `ai_summaries`/`notifications` 等集合存在且字段正确（不希望清空数据），可执行：

```bash
cd "G:\项目管理软件\v2\scripts"
node diagnose_and_fix_ai.mjs
```

> 以上脚本都需要你设置环境变量：`PB_URL`、`PB_ADMIN_EMAIL`、`PB_ADMIN_PASSWORD`（详见 `v2/frontend/env.example.txt`）。

---

## 四、前端重新打包并部署（可选）

如果你改了前端代码，并要更新服务器上的静态页面：

1. 在本地打包：

   ```bash
   cd "G:\项目管理软件\v2\frontend"
   npm run build
   ```

   打包结果会生成到 `v2/frontend/dist` 目录。

2. 用宝塔 / SFTP 将 `dist` 目录内容上传到服务器 Nginx 对应站点根目录（当前已经配置过一次，后续只需覆盖）。

## 五、常用命令速查表

| 场景 | 命令 |
|------|------|
| 本地启动后端 | `cd v2\\backend && .\\pocketbase.exe serve --http="0.0.0.0:8090"` |
| 本地启动前端 | `cd v2\\frontend && npm run dev` |
| 服务器重启 PocketBase | `cd /www/server/pocketbase && nohup ./pocketbase serve --http="0.0.0.0:8090" > pb.log 2>&1 &` |
| 重建演示数据（会清空） | `cd v2\\scripts && node database_rebuild.mjs` |
| 补齐 AI 集合（不清空） | `cd v2\\scripts && node diagnose_and_fix_ai.mjs` |
| 本地前端打包 | `cd v2\\frontend && npm run build` |

以后你只要记住：  
**“本地：backend + frontend；需要演示数据就跑 scripts；服务器：pocketbase 一个命令 +（可选）scripts 修复/重建”**，就能快速让整套系统跑起来。
