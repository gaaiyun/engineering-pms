## PR 详情

### PR 1：通知一期收尾（v2.96 → v2.97）

**问题：** 通知触发逻辑散落在 Home.tsx / Notifications.tsx 内联代码中，难以维护。
**方案：** 抽离为全局 Hook `useNotificationAlerts`，在 `<GlobalNotificationProvider />` 内挂载一次。
**影响文件：** 11 个修改 + 1 个新建（useNotificationAlerts.ts）+ 5 项单元测试
**验证：** npm test 通过，APK 编译通过

### PR 2：Android 前台服务 + PocketBase Realtime SSE（v2.97 → v2.98）

**核心决策：** 不用 Firebase/FCM，改用 PocketBase 自带的 Realtime SSE + Android 原生 Foreground Service 保活长连接。

**原因（详见 [research/2026-05-16-pr2-tech-reference.md](./superpowers/research/2026-05-16-pr2-tech-reference.md)）：**
- 避开 google-services.json + Play 审核
- 国内 FCM 网络不稳
- PocketBase Realtime 是现成基础设施，0 后端代码

**实现：**
- **Android 原生**：3 个新 Java 文件
  - `PbSseClient.java`（189 行）— OkHttp `okhttp-sse` 4.12 + 指数退避重连 1/2/4/8/16/30s + ±20% jitter
  - `RealtimeService.java`（203 行）— `dataSync` 类型前台服务，IMPORTANCE_LOW 持久通知，监听 ConnectivityManager
  - `RealtimePlugin.java`（106 行）— Capacitor 5 plugin，notifyListeners 转 JS
- **JS 端**：
  - `realtime.ts`（24 行）— Capacitor plugin 类型定义
  - `realtimeBridge.ts`（120 行）— React 集成，pb.authStore.onChange 联动 start/stop/updateToken
  - `App.tsx`：挂载 `<RealtimeBridgeProvider />`
- **服务端**：`backend/pb_hooks/realtime.pb.js` 设 idleTimeout=30 分钟
- **权限**：FOREGROUND_SERVICE / FOREGROUND_SERVICE_DATA_SYNC / WAKE_LOCK / RECEIVE_BOOT_COMPLETED / REQUEST_IGNORE_BATTERY_OPTIMIZATIONS / ACCESS_NETWORK_STATE
- **文档**：[notification-push-phase2.md](./notification-push-phase2.md) 整体重写；[android-background-keepalive.md](./android-background-keepalive.md) 11 家 OEM 保活引导

**验证：** gradle assembleDebug 编译通过，v2.98 APK 已生成。**实机后台验证需用户做**（详见 manual-qa）。

### PR 3：响应式 AppShell（桌面布局地基）

**方案：** 新建 `<AppShell>` 在路由根包裹所有受保护页面，通过 `useBreakpoint()` 切换：
- < 768px：透传（移动端保持现状）
- 769-1023px：折叠 Sidebar（64px 图标）
- ≥1024px：完整 Sidebar（240px）+ 顶栏（56px）

**新建组件：** Sidebar + TopBar + AppShell + useBreakpoint hook（CSS Grid 实现，0 新依赖）
**单元测试：** 10 项（5 useBreakpoint + 5 AppShell）

### PR 4：任务表格视图 + 批量操作（v3.0 桌面生产力）

**方案：** 桌面端 MyTasks 切换到 `@tanstack/react-table` 表格视图：
- 7 列（选择/序号/标题/项目/负责人/状态/截止日）
- 列头排序、多选、行点击进详情
- 底部黑色 Bulk Bar：批量改完成 / 批量删除
- 移动端保留卡片视图

**新依赖：** `@tanstack/react-table@^8.21.3`（bundle +~50KB gzip）
**单元测试：** 7 项

### PR 5：看板桌面体验（v3.0 视觉提升）

- KanbanBoard 桌面端隐藏冗余 NavBar，改用轻量 header
- drag-over 状态加 1.4s 脉冲呼吸动画 + 32px 光晕

### PR 6：文档收尾

- 重写 `notification-push-phase2.md` v3.0 架构
- 新增 `android-background-keepalive.md` 11 家 OEM 引导
- 本文档（v3.0 changelog）
- 更新 README
