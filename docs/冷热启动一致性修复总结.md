# 冷热启动一致性与横屏重构修复总结

**修复日期**: 2026-03-03  
**版本**: v2.2.0

## 一、问题背景

在实际使用中发现以下问题：
1. **冷/热启动行为不一致**：用户登录后，冷启动和热启动可能进入不同的页面
2. **返回键逻辑混乱**：从业务页面返回时可能意外回到登录页
3. **横屏体验差**：ProjectTimeline 使用 `transform: rotate(90deg)` 导致点击错位、滚动不自然
4. **缓存导致旧版本**：Service Worker 缓存策略可能导致更新后仍使用旧版路由逻辑

---

## 二、修复内容

### 1. 路由与导航一致性修复

#### 1.1 DefaultRedirect 组件
**文件**: `frontend/src/App.tsx`

**修复前**:
```tsx
const DefaultRedirect = () => {
  if (!pb.authStore.isValid) return <Navigate to="/app" />
  const role = pb.authStore.model?.role?.toLowerCase()
  if (role === 'admin' || role === 'manager') {
    return <Navigate to="/admin" />
  }
  return <Navigate to="/app" />
}
```

**修复后**:
```tsx
const DefaultRedirect = () => {
  if (!pb.authStore.isValid) return <Navigate to="/login" replace />
  const role = pb.authStore.model?.role?.toLowerCase()
  if (role === 'admin' || role === 'manager') {
    return <Navigate to="/admin" replace />
  }
  return <Navigate to="/app" replace />
}
```

**关键改进**:
- ✅ 所有导航都使用 `replace: true`，防止回退到登录页
- ✅ 未登录时正确跳转到 `/login` 而非 `/app`
- ✅ 确保冷/热启动行为一致

#### 1.2 登录成功导航
**文件**: `frontend/src/pages/Login.tsx`

**修复前**:
```tsx
if (role === 'admin' || role === 'manager') {
  navigate('/admin')
} else {
  navigate('/app')
}
```

**修复后**:
```tsx
if (role === 'admin' || role === 'manager') {
  navigate('/admin', { replace: true })
} else {
  navigate('/app', { replace: true })
}
```

**关键改进**:
- ✅ 登录成功后使用 `replace`，用户无法通过返回键回到登录页

#### 1.3 退出登录逻辑统一
**文件**: 
- `frontend/src/pages/admin/AdminDashboard.tsx`
- `frontend/src/pages/Profile.tsx`

**修复前**:
```tsx
const handleLogout = () => {
  pb.authStore.clear()
  navigate('/login')
}
```

**修复后**:
```tsx
const handleLogout = () => {
  pb.authStore.clear()
  localStorage.removeItem('rememberMe')
  sessionStorage.removeItem('pocketbase_auth')
  navigate('/login', { replace: true })
}
```

**关键改进**:
- ✅ 清理所有认证相关存储（localStorage + sessionStorage）
- ✅ 使用 `replace` 防止退出后回退到业务页面
- ✅ 确保下次启动时不会自动登录（除非勾选"记住我"）

#### 1.4 AdminDashboard Tab 验证
**文件**: `frontend/src/pages/admin/AdminDashboard.tsx`

**新增逻辑**:
```tsx
useEffect(() => {
  if (tabFromUrl && !VALID_TABS.includes(tabFromUrl)) {
    setSearchParams({ tab: 'dashboard' }, { replace: true })
  }
}, [tabFromUrl, setSearchParams])
```

**关键改进**:
- ✅ 检测无效 tab 参数并自动重定向到 dashboard
- ✅ 使用 `replace` 避免历史记录污染

### 2. 横屏功能重构

#### 2.1 移除旋转方案
**文件**: `frontend/src/pages/ProjectTimeline.tsx`

**修复前**:
```tsx
const [isLandscape, setIsLandscape] = useState(false)
const isFullscreen = !isPC && isLandscape

const landscapeStyle: React.CSSProperties = isFullscreen ? {
  position: 'fixed',
  top: 0,
  left: 0,
  width: '100vh',
  height: '100vw',
  transform: 'rotate(90deg)',
  transformOrigin: 'top left',
  marginLeft: '100vw',
  zIndex: 9999,
  overflow: 'hidden',
} : {}
```

**修复后**:
```tsx
const [isNaturalLandscape, setIsNaturalLandscape] = useState(
  window.innerWidth > window.innerHeight
)

// 横屏时自动放大单元格宽度以利用更多水平空间
const landscapeMultiplier = isNaturalLandscape && !isPC ? 1.3 : 1.0
const rawCellWidth = BASE_CELL_WIDTH * scale * landscapeMultiplier
```

**关键改进**:
- ✅ 移除 `transform: rotate(90deg)` 及相关复杂逻辑
- ✅ 使用自然横屏检测（`window.innerWidth > window.innerHeight`）
- ✅ 横屏时自动放大单元格宽度 1.3 倍，充分利用水平空间
- ✅ 移除强制横屏按钮，改为自动适配

#### 2.2 CSS 清理
**文件**: `frontend/src/index.css`

**移除的样式**:
```css
.landscape-mode {
  position: fixed !important;
  transform: rotate(90deg);
  /* ... 大量旋转相关样式 */
}
```

**新增的样式**:
```css
/* === Timeline Responsive Layout === */
.timeline-container {
  overflow: hidden;
  background: #f8fafc;
}

/* Landscape optimizations via media query */
@media (orientation: landscape) and (max-width: 1024px) {
  .timeline-container .timeline-header {
    padding: 12px 16px !important;
  }
  
  .timeline-container .timeline-scroll-area {
    overscroll-behavior: contain;
  }
}
```

**关键改进**:
- ✅ 使用标准 CSS 媒体查询 `@media (orientation: landscape)`
- ✅ 移除所有旋转相关的 hack 代码
- ✅ 更简洁、更可维护的响应式方案

### 3. Service Worker 缓存策略优化

#### 3.1 版本更新
**文件**: `frontend/public/sw.js`

**修复前**:
```js
const CACHE_NAME = 'engineering-pms-v2.1.0';
const STATIC_ASSETS = [
  '/',

  '/manifest.json',
];
```

**修复后**:
```js
const CACHE_NAME = 'engineering-pms-v2.2.0';

  '/',
  '/manifest.json',
];
```

**关键改进**:
- ✅ 更新版本号到 v2.2.0
- ✅ 移除 `/index.html` 从静态缓存列表（改为动态策略）

#### 3.2 index.html 缓存策略
**文件**: `frontend/public/sw.js`

**新增逻辑**:
```js
// index.html 始终从网络获取，避免缓存旧版路由逻辑
if (url.pathname === '/' || url.pathname === '/index.html') {
  event.respondWith(
    fetch(event.request)
      .catch(() => caches.match('/index.html'))
  );
  return;
}
```

**关键改进**:
- ✅ index.html 始终尝试从网络获取最新版本
- ✅ 仅在网络失败时才使用缓存
- ✅ 防止旧版路由逻辑导致冷/热启动不一致

## 五、部署建议

### 5.1 前端部署
```bash
cd frontend
npm run build
# 将 dist/ 目录部署到服务器
```

### 5.2 Android 打包
```bash
cd frontend

npx cap sync android

# 在 Android Studio 中打包 APK/AAB
```

### 5.3 强制更新 Service Worker
部署新版本后，建议在服务器配置中添加：
```nginx
# 禁用 index.html 缓存
location = /index.html {
    add_header Cache-Control "no-cache, no-store, must-revalidate";
}
```

或在用户端清理缓存：
- 设置页面 → 清除缓存
- 或在浏览器中强制刷新（Ctrl+Shift+R）

## 六、后续优化建议

### 6.1 Android 返回键增强
当前代码已预留 Capacitor App 插件集成点，建议后续添加：
```tsx
import { App } from '@capacitor/app';

useEffect(() => {
  const handler = App.addListener('backButton', ({ canGoBack }) => {
    if (!canGoBack || window.location.pathname === '/admin' || window.location.pathname === '/app') {
      App.exitApp();
    } else {
      navigate(-1);
    }
  });
  return () => handler.remove();
}, [navigate]);
```

### 6.2 横屏锁定（可选）
如需强制锁定横屏，可使用 Capacitor ScreenOrientation 插件：
```tsx
import { ScreenOrientation } from '@capacitor/screen-orientation';

// 进入时间轴时锁定横屏
await ScreenOrientation.lock({ orientation: 'landscape' });

// 离开时恢复
await ScreenOrientation.unlock();
```

### 6.3 版本号自动化
建议在构建时自动注入版本号：
```js
// vite.config.ts
import { defineConfig } from 'vite'
import { version } from './package.json'

export default defineConfig({
  define: {
    __APP_VERSION__: JSON.stringify(version)
  }
})
```
