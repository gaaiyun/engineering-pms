## 1. 摘要与背景

### 1.1 产品摘要

本产品是一款专为**工程管理公司**打造的**全流程协同管理平台**。通过"三步法"方法论（标准化拆解、可视化追踪、流程化配合）解决多项目并行场景下的**信息断层与协作内耗**问题。

**核心特点**：
- 📱 **移动优先**：以 Android APK + Web 双端覆盖
- 🎯 **角色聚焦**：老板看风险、经理管流程、员工做执行
- 🔄 **闭环设计**：强制"完成即交接"，杜绝推诿扯皮
- 🤖 **AI 赋能**：智能风险诊断与问答助手

### 1.2 项目背景

#### 1.2.1 行业现状

工程管理行业长期面临以下痛点：

1. **信息不透明**：老板依赖口头汇报，无法实时掌握项目进度
2. **协作效率低**：多项目并行时，跨部门配合缺乏有效机制
3. **责任不清晰**：任务流转时缺少交接记录，出问题难追责
4. **工具不适配**：现有项目管理软件（如钉钉、飞书）偏通用，无法满足工程特有流程

#### 1.2.2 竞品分析

| 竞品 | 优势 | 劣势 | 差异化机会 |
|------|------|------|-----------|
| **钉钉项目** | 生态完整、用户基数大 | 偏通用、工程适配性差 | 垂直深耕工程领域 |
| **Teambition** | 交互友好、看板成熟 | 缺少工程特有功能 | 强交接闭环 |
| **广联达** | 工程领域专业 | 系统复杂、学习成本高 | 轻量化+移动优先 |
| **Excel 表格** | 灵活、无成本 | 无法协同、数据孤岛 | 实时协作+AI |

#### 1.2.3 市场机会

- 工程管理数字化转型需求旺盛
- 中小型工程公司缺乏低成本、易用的管理工具
- 移动端场景（工地、外勤）需求突出
- AI 技术成熟，可赋能管理决策

### 1.3 产品愿景

> **让每个工程项目都能被高效管理**

三年愿景：
- Y1：单公司内部部署，验证 PMF（Product-Market Fit）
- Y2：支持多租户 SaaS，服务 100+ 工程公司
- Y3：构建工程管理标准化能力库，成为行业基础设施

### 2.2 "三步法"方法论

本系统的核心理念是"三步法"：

#### 第一步：标准化拆解

| 拆解维度 | 说明 | 示例 |
|----------|------|------|
| **阶段拆解** | 将项目按阶段划分 | 前期准备 → 主体施工 → 竣工验收 |
| **任务拆解** | 将阶段细分为任务节点 | 图纸审核、材料送检、基础浇筑 |
| **时限拆解** | 明确每个任务的时限 | 开始日期、截止日期、里程碑 |
| **责任拆解** | 明确每个任务的负责人 | 单人负责 / 多人协作 |

#### 第二步：可视化追踪

| 视图类型 | 适用场景 | 信息密度 |
|----------|----------|----------|
| **看板视图** | 日常状态跟踪 | 中（聚焦状态） |
| **时间轴视图** | 进度规划与复盘 | 高（时间维度） |
| **列表视图** | 批量操作与筛选 | 高（数据完整） |
| **仪表盘视图** | 管理层决策 | 低（关键指标） |

#### 第三步：流程化配合

```
执行层（员工）                管理层（经理）              决策层（老板）
     │                            │                          │
     │  1. 执行任务               │                          │
     │                            │                          │
     │  2. 遇到卡点 ─────────────▶│  3. 24h响应卡点          │
     │                            │                          │
     │                            │  4. 协调资源             │
     │                            │                          │
     │  5. 完成任务               │                          │
     │                            │                          │
     │  6. 提交交接 ─────────────▶│  7. 审核交接             │
     │                            │                          │
     │                            │  8. 分配新任务           │
     │                            │                          │
     │                            │  9. 超时卡点升级 ───────▶│ 10. 决策干预
     │                            │                          │
```

#### 2.3.3 交接规则

1. **必填规则**：完成任务时必须提交交接提案
2. **提案内容**：下一步任务名称、负责人、截止日期、描述
3. **审核时效**：经理应在 24 小时内审核
4. **通过后果**：系统自动创建新任务、通知新负责人
5. **驳回后果**：退回修改，记录驳回原因

#### 2.3.4 卡点规则

1. **必填字段**：卡点原因类型、详细描述、需要谁配合
2. **通知规则**：自动通知项目经理
3. **超时升级**：卡点超过 24 小时未解决，升级到老板
4. **解决记录**：解决时需填写解决方案

### 5.2 功能详细描述

#### 5.2.1 项目看板（Kanban）

**功能描述**：
以四列（或五列）看板形式展示项目任务，支持拖拽流转任务状态。

**功能要点**：
| 要点 | 描述 |
|------|------|
| **列定义** | 待开始、进行中、卡点、已逾期、已完成 |
| **任务卡片** | 显示：任务名、负责人、截止日、优先级标签 |
| **拖拽交互** | 鼠标拖拽（PC）/ 长按 250ms 触发（移动端） |
| **即时更新** | 乐观更新，拖拽后立即反映在 UI |
| **排序规则** | 列内按 `sequence` 字段排序 |
| **筛选能力** | 按负责人筛选（规划中） |

**交互规范**：
- 拖拽触发：移动端长按 250ms，PC 端直接拖动
- 拖拽反馈：被拖拽卡片透明度降低、显示占位符
- 放置反馈：目标位置高亮
- 失败回滚：网络失败时自动回滚到原位置

**技术实现**：
- 基于 `@dnd-kit` 库
- 使用 `SortableContext` 管理列内排序
- 使用 `DndContext` 管理跨列拖拽
- 乐观更新通过 `TanStack Query` 的 `onMutate` 实现

#### 5.2.2 项目时间轴（甘特图）

**功能描述**：
以时间轴形式可视化项目任务进度，支持缩放、滚动、分组。

**功能要点**：
| 要点 | 描述 |
|------|------|
| **时间刻度** | 日/周/月切换（规划中） |
| **任务条** | 显示任务名、状态颜色、进度 |
| **缩放能力** | 50%-200%，双指捏合（移动端） |
| **人员分组** | 按负责人分组显示 |
| **详情查看** | 点击任务条弹出详情抽屉 |

**交互规范**：
- 横向滚动：手势滑动 / 鼠标滚轮
- 缩放：双指捏合 / 缩放按钮
- 今日线：红色虚线标记当前日期
- 状态颜色：与看板状态颜色一致

#### 5.2.3 交接提案

**功能描述**：
员工完成任务时必须提交交接提案，由经理审核后系统自动创建新任务。

**表单字段**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 下一步任务名称 | 文本 | ✅ | 清晰描述下一步工作 |
| 任务描述 | 长文本 | ❌ | 详细说明 |
| 负责人 | 用户选择 | ✅ | 可多选 |
| 截止日期 | 日期 | ✅ | 建议日期 |
| 完成情况说明 | 长文本 | ❌ | 本任务完成情况 |

**审核流程**：
```
员工提交交接 → 经理收到通知 → 进入审核中心
                                    │
                    ┌───────────────┴───────────────┐
                    │                               │
                 通过                            驳回
                    │                               │
                    ▼                               ▼
             系统创建新任务                   退回给员工
             通知新负责人                     填写驳回原因
             记录审计日志                     员工修改后重新提交
```

#### 5.2.4 卡点上报

**功能描述**：
员工遇到问题无法继续时，通过卡点上报机制请求协助。

**表单字段**：
| 字段 | 类型 | 必填 | 选项 |
|------|------|------|------|
| 卡点原因类型 | 单选 | ✅ | 等待审批、等待材料、技术问题、外部因素、其他 |
| 详细描述 | 长文本 | ✅ | 具体说明遇到的问题 |
| 需要谁配合 | 用户选择 | ❌ | 选择需要协助的人 |
| 期望解决时间 | 日期 | ❌ | 期望问题解决的时间 |

**通知规则**：
1. 卡点上报后，立即通知项目经理
2. 被 @ 的协助人也收到通知
3. 卡点超过 24 小时未解决，升级通知老板

#### 5.2.5 AI 智能助手

**功能描述**：
基于 SiliconFlow（DeepSeek）提供智能分析能力。

**AI 简报**：
- 自动聚合项目数据（任务、卡点、审计日志）
- 调用 AI 生成 Markdown 格式简报
- 包含：整体评估、进展、卡点分析、风险预警、建议
- 保存到 `ai_summaries` 集合

**AI 问答**：
- 支持多轮对话
- 上下文感知（基于项目数据）
- 支持问题：
  - "当前最大的风险是什么？"
  - "哪些任务逾期了？"
  - "张三现在负责哪些任务？"
  - "这个项目的卡点主要是什么原因？"

**技术规格**：
| 配置项 | 值 |
|--------|-----|
| AI 提供商 | SiliconFlow |
| 模型 | DeepSeek-V3 |
| API 端点 | `https://api.siliconflow.cn/v1/chat/completions` |
| 响应格式 | JSON（简报）/ 文本（问答） |
| 超时时间 | 30 秒 |

### 7.2 核心用例

#### UC-01: 员工完成任务并交接

**用例名称**：任务完成与交接

**参与者**：员工、经理

**前置条件**：
- 员工已登录系统
- 员工有进行中的任务

**主流程**：
1. 员工打开"我的任务"
2. 员工选择要完成的任务
3. 系统显示任务详情
4. 员工点击"完成任务"按钮
5. 系统弹出交接提案表单
6. 员工填写：下一步任务名称、负责人、截止日期
7. 员工提交交接提案
8. 系统将任务状态改为"已完成"
9. 系统创建交接提案记录
10. 系统通知项目经理
11. 经理收到通知，进入审核中心
12. 经理查看交接详情
13. 经理点击"通过"
14. 系统自动创建新任务
15. 系统通知新负责人

**异常流程**：
- 6a. 员工取消提交 → 返回任务详情
- 13a. 经理驳回 → 填写驳回原因 → 通知员工修改

**后置条件**：
- 原任务状态变为"已完成"
- 新任务被创建
- 新负责人收到通知
- 审计日志记录操作

## 9. 交互设计

### 9.1 设计原则

| 原则 | 描述 | 实践 |
|------|------|------|
| **移动优先** | 以移动端体验为主 | 大按钮、大卡片、触控友好 |
| **即时反馈** | 操作后立即响应 | 乐观更新、Loading 状态 |
| **减少输入** | 降低输入成本 | 选择器、快捷操作 |
| **信息层级** | 重要信息突出 | 颜色、大小、位置 |
| **一致性** | 交互逻辑统一 | 统一的操作模式 |

### 9.2 核心交互

#### 拖拽交互（看板）
- **触发**：移动端长按 250ms / PC 端直接拖动
- **视觉反馈**：被拖拽卡片半透明、显示占位符
- **放置反馈**：目标区域高亮
- **完成反馈**：卡片归位动画 + Toast 提示

#### 状态切换
- **视觉**：按钮颜色变化、图标变化
- **反馈**：触感反馈（移动端）、Toast 提示
- **回滚**：失败时自动回滚、显示错误原因

#### 表单提交
- **验证**：实时验证、提交前二次验证
- **Loading**：按钮 Loading 状态
- **成功**：Toast 提示 + 页面跳转/关闭弹窗
- **失败**：Toast 错误提示、保留输入内容

### 9.3 响应式设计

| 断点 | 宽度 | 布局 |
|------|------|------|
| Mobile | < 768px | 单列、底部 Tab |
| Tablet | 768-1024px | 双列（规划中） |
| Desktop | > 1024px | 多列、侧边栏 |
