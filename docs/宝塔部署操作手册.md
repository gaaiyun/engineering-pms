## 一、本地开发

> 前提：已在 `frontend` 和 `scripts` 目录各执行过一次 `npm install`。

### 1. 启动 PocketBase 后端

在 PowerShell 中：

```bash
cd "G:\项目管理软件_v2\backend"
.\pocketbase.exe serve --http="0.0.0.0:8090"
```

- 端口 `8090` 与线上保持一致，终端不要关。
- 也可以直接双击同目录下的 `启动后端.bat`。

### 2. 启动前端（Vite + React）

新开一个终端：

```bash
cd "G:\项目管理软件_v2\frontend"
npm run dev
```

浏览器访问 `http://localhost:5173`。

默认账号（由初始化脚本创建）：

| 角色 | 账号 | 密码 |
|------|------|------|
| 老板/管理员 | `admin_boss` | `12345678` |
| 经理 | `zhang_manager` / `wang_manager` | `12345678` |
| 员工 | `chen_doc` / `li_audit` | `12345678` |

> 若未运行初始化脚本，账号以你在 PocketBase 管理后台创建的为准。

### 3. 初始化 / 重建数据

以下脚本都需要环境变量 `PB_URL`、`PB_ADMIN_EMAIL`、`PB_ADMIN_PASSWORD`（详见 `frontend/env.example.txt`）。

**一键重建演示数据（⚠️ 会清空业务数据）：**

```bash
cd "G:\项目管理软件_v2\scripts"
node database_rebuild.mjs
```

会清空并重建 `tasks` / `projects` / `handoffs` / `comments` / `audit_logs` / `ai_summaries` / `notifications` 等集合。请勿在生产库上执行。

**仅补齐 AI 相关集合（不清空数据，适合线上）：**

```bash
cd "G:\项目管理软件_v2\scripts"
node diagnose_and_fix_ai.mjs
```

### 4. 前端打包

```bash
cd "G:\项目管理软件_v2\frontend"
npm run build
```

产物位于 `frontend/dist`，可部署到服务器或用于 APK 打包。

### 5. 常用命令速查

| 场景 | 命令 |
|------|------|
| 本地启动后端 | `cd backend && .\pocketbase.exe serve --http="0.0.0.0:8090"` |
| 本地启动前端 | `cd frontend && npm run dev` |
| 重建演示数据（会清空） | `cd scripts && node database_rebuild.mjs` |
| 补齐 AI 集合（不清空） | `cd scripts && node diagnose_and_fix_ai.mjs` |
| 前端打包 | `cd frontend && npm run build` |

## 二、服务器部署（宝塔）

### 1. 准备工作

1. **下载 Linux 版 PocketBase**
   - 服务器是 Linux 系统，不能用本地的 Windows 版 exe。
   - 下载: [pocketbase_0.26.3_linux_amd64.zip](https://github.com/pocketbase/pocketbase/releases/download/v0.26.3/pocketbase_0.26.3_linux_amd64.zip)
   - 解压后得到一个无后缀的 `pocketbase` 文件。

2. **构建前端**
   - 前端支持 **PocketBase 地址自动推导**（见 `frontend/src/lib/pocketbase.ts`）：
     - http 站点 → `http://当前域名:8090`
     - https 站点 → `https://当前域名/pb`（需 Nginx 反代，避免 Mixed Content）
   - 也可通过 `VITE_PB_URL` 显式指定（适合 App 打包 / 多环境）。
   - 无需手动改 `pocketbase.ts`，直接构建即可：
     ```bash
     cd "G:\项目管理软件_v2\frontend"
     npm run build
     ```
   - 使用 https 时建议配合下文的 **/pb 反代** 方案。

### 2. 后端部署（PocketBase）

#### 上传文件

1. 宝塔面板 → **文件** → 进入 `/www/server`，新建文件夹 `pocketbase`。
2. 进入 `/www/server/pocketbase`，上传 Linux 版 `pocketbase` 文件。
3. 右键 `pocketbase` → **权限** → 勾选 **执行** (755) → 确定。

#### 启动服务

宝塔 → **终端**，执行：

```bash
cd /www/server/pocketbase
nohup ./pocketbase serve --http="0.0.0.0:8090" > pb.log 2>&1 &
```

验证：浏览器访问 `http://<YOUR_PB_HOST>:8090/_/` 看到登录界面即成功。

> 记得在阿里云安全组里**放行 8090 端口**。

### 3. 前端部署（React）

#### 创建站点

1. 宝塔 → **网站** → **添加站点**。
2. **域名**：填写你的 IP 或域名。
3. **根目录**：`/www/wwwroot/project_v2`（没有就新建）。
4. **PHP 版本**：纯静态，选"不使用 PHP"。

#### 上传前端代码

将本地 `frontend/dist` 文件夹内的**所有内容**上传到 `/www/wwwroot/project_v2`，目录结构：

```
/www/wwwroot/project_v2/
├── index.html
└── assets/
```

#### 配置 Nginx

**解决 SPA 刷新 404：**

网站 → 点击站点 → **配置文件**，在 `server { }` 块中添加：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

**（推荐）配置 /pb 反代（上 https 必做）：**

```nginx
location /pb/ {
    proxy_pass http://127.0.0.1:8090/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
```

> 配置后可将 8090 端口仅对内网开放（更安全），前端通过 `https://你的域名/pb` 访问后端。

### 4. 数据迁移（本地 → 服务器，仅初次部署）

1. 本地 `backend/pb_data/data.db` 上传到服务器 `/www/server/pocketbase/pb_data/`，覆盖同名文件。
2. 重启后端：
   ```bash
   pkill pocketbase
   cd /www/server/pocketbase
   nohup ./pocketbase serve --http="0.0.0.0:8090" > pb.log 2>&1 &
   ```

### 5. 完成验证

- 浏览器访问 `http://<YOUR_DOMAIN_OR_IP>` 验证网页版。
- 可使用域名/IP 重新打包 Android App 分发给同事。

## ⭐ v3.04 增量升级流程（**已上线后再发版用这个**）

> 适用场景：服务器已经在跑 v2.x 或 v3.x 老版本，需要升级到 v3.04 的最新代码（含 6 个 PB hooks + 3 个新 migrations + 前端代码）。**默认情况只需要上传 frontend/dist** — 后端代码（hooks/migrations）变化时才需要后端步骤。

### 0. 升级前必备

1. **本地构建一次最新前端：**
   ```bash
   cd "G:\项目管理软件_v2\frontend"
   npm run build
   ```
   产物在 `frontend/dist/`。
2. **服务器备份（不可跳）：**
   ```bash
   cd /www/server/pocketbase
   DATE=$(date +%Y%m%d_%H%M)
   cp -r pb_data backups/pb_data_${DATE}_pre_upgrade
   [ -d pb_hooks ] && cp -r pb_hooks backups/pb_hooks_${DATE}_pre_upgrade
   ```

### 1. 上传前端（**最常见场景：只发前端**）

**宝塔文件管理器（最简单）：**
1. 登录宝塔 → **文件** → 进入 `/www/wwwroot/<你的站点目录>`（如 `project_v2`）
2. **删除旧文件**：选中 `index.html`、`assets/`、`icons/`、`manifest.json`、`sw.js` 全部删除
3. **上传新 dist**：把本地 `frontend/dist/` 内的**所有内容**（不是 dist 文件夹本身）拖到此目录
4. 验证目录结构：
   ```
   /www/wwwroot/project_v2/
   ├── index.html
   ├── assets/
   ├── icons/
   ├── manifest.json
   └── sw.js
   ```

**或用 SCP 命令行（更快）：**
```bash
# 本地 PowerShell / Git Bash 执行（替换 root@<IP>）
scp -r G:/项目管理软件_v2/frontend/dist/* root@<服务器IP>:/www/wwwroot/project_v2/
```

**或 rsync（最稳，增量 + 自动删除旧文件）：**
```bash
rsync -avz --delete \
  G:/项目管理软件_v2/frontend/dist/ \
  root@<服务器IP>:/www/wwwroot/project_v2/
```

完成后浏览器**强刷**（Ctrl+Shift+R）一次，新版生效。**不需要重启 PocketBase**。

### 2. 上传后端 hooks（**仅当本地 `backend/pb_hooks/` 有变化时**）

如果你只是改了前端代码（如 UI/组件/页面），跳过本步骤。
如果你的 `backend/pb_hooks/` 添加了新 .js 文件或修改了现有 hook 内容，做：

```bash
# 本地命令：上传 5 个 hooks 到服务器
scp G:/项目管理软件_v2/backend/pb_hooks/*.js root@<IP>:/www/server/pocketbase/pb_hooks/
```

宝塔文件管理器方式：进入 `/www/server/pocketbase/pb_hooks/`，把本地 `backend/pb_hooks/` 下 **5 个 .js 文件**全部上传覆盖：
- `realtime.pb.js`
- `handoffs_status_sync.pb.js`
- `audit_logs_reject_sync.pb.js`
- `project_progress_sync.pb.js`
- `llm_proxy.pb.js`

### 3. 上传后端 migrations（**仅当本地 `backend/pb_migrations/` 有新文件时**）

```bash
# 本地命令：上传 38 个 migrations（PB 启动时自动跳过已应用的）
scp G:/项目管理软件_v2/backend/pb_migrations/*.js root@<IP>:/www/server/pocketbase/pb_migrations/
```

### 4. 应用 migrations + 重启 PB（**仅当做了 step 2 或 step 3 时**）

```bash
ssh root@<服务器IP>
cd /www/server/pocketbase

# 应用未执行的 migration
./pocketbase migrate up

# 重启（先杀旧进程，再后台启）
pkill -f "pocketbase serve" || true
sleep 2
nohup ./pocketbase serve --http="0.0.0.0:8090" > pb.log 2>&1 &
echo "PB started, PID=$!"

# 验证起来 + 看 hook 加载状态
sleep 3
curl -s http://127.0.0.1:8090/api/health
echo ""
tail -30 pb.log
```

期望看到 `Server started at http://0.0.0.0:8090`。pb.log 不应有 hook 加载报错。

### 5. （首次启用 AI 才做）配置 LLM API key

如果用户要使用 AI 功能（C1 已改为服务端代理），需要在 PB Admin UI 新建一条 `app_settings` record：

1. 浏览器访问 `http://<服务器IP>:8090/_/`（PB Admin UI）
2. PB superuser 登录
3. **Collections** → 选 `app_settings` → **+ New record**：
   - `key`: `siliconflow_api_key`
   - `value`: 你的 sk-xxxx... 真实密钥
   - `description`: `SiliconFlow LLM API key`
4. Save

之后前端 AI Console / chatWithAI 自动走 `POST /api/custom/llm-proxy`，浏览器侧不再接触 API key。

### 6. 升级后验证（3 分钟）

```bash
# 1. 前端访问
curl -s -o /dev/null -w "Frontend HTTP %{http_code}\n" http://<服务器IP>/

# 2. PB API
curl -s http://<服务器IP>:8090/api/health

# 3. 浏览器实测
# - 登录任一账号
# - 看到管理控制台或员工首页
# - 进入"我的任务"看到表格视图（桌面）或卡片视图（手机）
# - 进"审核中心 ?tab=handoff" 看到 handoff 列表
```

### 7. 回滚（万一新版出问题）

```bash
ssh root@<服务器IP>
cd /www/server/pocketbase

# 还原数据库
pkill -f "pocketbase serve"
cp -r backups/pb_data_<DATE>_pre_upgrade/* pb_data/

# 还原 hooks（如有上传新的）
rm -f pb_hooks/handoffs_status_sync.pb.js
rm -f pb_hooks/audit_logs_reject_sync.pb.js
rm -f pb_hooks/project_progress_sync.pb.js
rm -f pb_hooks/llm_proxy.pb.js

# 重启
nohup ./pocketbase serve --http="0.0.0.0:8090" > pb.log 2>&1 &

# 前端：把上一版 dist 备份还原（如果之前备份了的话）
```

### v3.04 升级流程速查（一条龙）

最常用场景是**只更新前端 UI**，无须改后端：

```powershell
# 1. 本地构建
cd "G:\项目管理软件_v2\frontend"; npm run build

# 2. 上传 dist 到服务器（任一方式）
```

## 三、APK 打包

### 1. 前置准备

1. **Node.js**（已有）。
2. **Android Studio**：[下载地址](https://developer.android.com/studio)，安装后配置 SDK。
3. **Java JDK**：Android Studio 通常自带，或单独安装 JDK 17。

### 2. 构建 & 同步

```bash
cd "G:\项目管理软件_v2\frontend"
npm run build

npx cap add android   # 仅首次执行
npx cap sync          # 每次改代码后执行
```

### 3. 使用 Android Studio 打包

```bash
npx cap open android
```

1. 等待项目加载完成（首次需下载依赖，注意右下角进度条）。
2. **（可选）修改图标和名称**：替换 `app/src/main/res` 下的图标，修改 `strings.xml` 中的应用名称。
3. **测试运行**：手机开启开发者模式 + USB 调试，点击绿色 ▶️ 按钮。
4. **生成 APK**：
   - **Build** → **Generate Signed Bundle / APK** → 选择 **APK** → Next
   - 创建 Key store（记住密码），选择 Release
   - 产物位于 `android/app/release/*.apk`

### 4. 常见问题

#### 手机显示"无法连接服务器"

App 运行在手机上，需要能访问 PocketBase 地址。

**正式发布（推荐）**：通过环境变量指定 PocketBase 地址：

```powershell
setx VITE_PB_URL "http://<YOUR_PB_HOST>:8090"
```

重新打开终端后：

```bash
cd "G:\项目管理软件_v2\frontend"
npm run build
npx cap sync
```

**本地调试**：手机和电脑连同一 WiFi，将 `VITE_PB_URL` 设为电脑局域网 IP（如 `http://192.168.x.x:8090`）。

#### Android 9+ 禁止 HTTP 请求

在 `android/app/src/main/AndroidManifest.xml` 的 `<application>` 标签中添加：

```xml
android:usesCleartextTraffic="true"
```

> 建议正式环境使用 https + /pb 反代，仅内测阶段开启 cleartext。

## 四、自动备份配置

备份脚本位于 `scripts/auto_backup.mjs`，策略：每 12 小时备份一次，保留 60 天。

### 4.1 Linux 服务器（宝塔面板）

```bash
# 1. 确认脚本路径（假设项目部署在 /www/server/pocketbase/）
ls /www/server/pocketbase/scripts/auto_backup.mjs

# 2. 手动执行一次，确认正常
cd /www/server/pocketbase
PB_DATA_DIR=/www/server/pocketbase/pb_data \
BACKUP_DIR=/www/server/pocketbase/backups \
node scripts/auto_backup.mjs

# 3. 配置定时任务（每12小时自动运行）
crontab -e
# 在末尾添加以下一行：
0 */12 * * * cd /www/server/pocketbase && PB_DATA_DIR=/www/server/pocketbase/pb_data BACKUP_DIR=/www/server/pocketbase/backups node /www/server/pocketbase/scripts/auto_backup.mjs >> /var/log/pb_backup.log 2>&1

# 4. 确认 crontab 已保存
crontab -l
```

**宝塔面板方式**：也可以在宝塔 → 计划任务 → Shell脚本 中添加上述命令，设置每 12 小时执行。

### 4.2 Windows 开发机

```powershell
# 创建计划任务（以管理员运行 PowerShell）
schtasks /create /tn "PB_Backup" /tr "node G:\项目管理软件_v2\scripts\auto_backup.mjs" /sc hourly /mo 12 /f

# 查看计划任务
schtasks /query /tn "PB_Backup"

# 删除计划任务
schtasks /delete /tn "PB_Backup" /f
```

### 4.3 备份文件位置

| 环境 | 备份目录 | 保留时间 |
|---|---|---|
| Linux 服务器 | `/www/server/pocketbase/backups/` | 60 天 |
| Windows 开发机 | `G:\项目管理软件_v2\backups\` | 60 天 |

备份文件命名格式：`data_2026-03-04T10-00-00.db`（Linux 会自动 gzip 压缩为 `.db.gz`）

### 4.4 手动恢复

```bash
# 1. 停止 PocketBase
pkill -f pocketbase  # Linux
# 或手动关闭进程     # Windows

# 2. 替换数据库文件
cp /www/server/pocketbase/backups/data_2026-03-04T10-00-00.db /www/server/pocketbase/pb_data/data.db

# 3. 重启 PocketBase
cd /www/server/pocketbase
nohup ./pocketbase serve --http="0.0.0.0:8090" > pb.log 2>&1 &
```
