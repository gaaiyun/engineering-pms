## 一、本地开发

> 前提：已在 `frontend` 和 `scripts` 目录各执行过一次 `npm install`。

### 1. 启动 PocketBase 后端

在 PowerShell 中：

```bash
cd "G:\项目管理软件_v2\backend"
.\pocketbase.exe serve --http="0.0.0.0:8090"
```

- 端口 `8090` 与线上保持一致，终端不要关。
- 也可以直接双击同目录下的 `启动后端.bat`。

### 2. 启动前端（Vite + React）

新开一个终端：

```bash
cd "G:\项目管理软件_v2\frontend"
npm run dev
```

浏览器访问 `http://localhost:5173`。

默认账号（由初始化脚本创建）：

| 角色 | 账号 | 密码 |
|------|------|------|
| 老板/管理员 | `admin_boss` | `12345678` |
| 经理 | `zhang_manager` / `wang_manager` | `12345678` |
| 员工 | `chen_doc` / `li_audit` | `12345678` |

> 若未运行初始化脚本，账号以你在 PocketBase 管理后台创建的为准。

### 3. 初始化 / 重建数据

以下脚本都需要环境变量 `PB_URL`、`PB_ADMIN_EMAIL`、`PB_ADMIN_PASSWORD`（详见 `frontend/env.example.txt`）。

**一键重建演示数据（⚠️ 会清空业务数据）：**

```bash
cd "G:\项目管理软件_v2\scripts"
node database_rebuild.mjs
```

会清空并重建 `tasks` / `projects` / `handoffs` / `comments` / `audit_logs` / `ai_summaries` / `notifications` 等集合。请勿在生产库上执行。

**仅补齐 AI 相关集合（不清空数据，适合线上）：**

```bash
cd "G:\项目管理软件_v2\scripts"
node diagnose_and_fix_ai.mjs
```

### 4. 前端打包

```bash
cd "G:\项目管理软件_v2\frontend"
npm run build
```

产物位于 `frontend/dist`，可部署到服务器或用于 APK 打包。

### 5. 常用命令速查

| 场景 | 命令 |
|------|------|
| 本地启动后端 | `cd backend && .\pocketbase.exe serve --http="0.0.0.0:8090"` |
| 本地启动前端 | `cd frontend && npm run dev` |

| 前端打包 | `cd frontend && npm run build` |

## 二、服务器部署（宝塔）

### 1. 准备工作

1. **下载 Linux 版 PocketBase**
   - 服务器是 Linux 系统，不能用本地的 Windows 版 exe。
   - 下载: [pocketbase_0.26.3_linux_amd64.zip](https://github.com/pocketbase/pocketbase/releases/download/v0.26.3/pocketbase_0.26.3_linux_amd64.zip)
   - 解压后得到一个无后缀的 `pocketbase` 文件。

2. **构建前端**
   - 前端支持 **PocketBase 地址自动推导**（见 `frontend/src/lib/pocketbase.ts`）：
     - http 站点 → `http://当前域名:8090`
     - https 站点 → `https://当前域名/pb`（需 Nginx 反代，避免 Mixed Content）
   - 也可通过 `VITE_PB_URL` 显式指定（适合 App 打包 / 多环境）。
   - 无需手动改 `pocketbase.ts`，直接构建即可：
     ```bash
     cd "G:\项目管理软件_v2\frontend"
     npm run build
     ```
   - 使用 https 时建议配合下文的 **/pb 反代** 方案。

### 2. 后端部署（PocketBase）

#### 上传文件

1. 宝塔面板 → **文件** → 进入 `/www/server`，新建文件夹 `pocketbase`。
2. 进入 `/www/server/pocketbase`，上传 Linux 版 `pocketbase` 文件。
3. 右键 `pocketbase` → **权限** → 勾选 **执行** (755) → 确定。

#### 启动服务

宝塔 → **终端**，执行：

```bash
cd /www/server/pocketbase
nohup ./pocketbase serve --http="0.0.0.0:8090" > pb.log 2>&1 &
```

验证：浏览器访问 `http://<YOUR_PB_HOST>:8090/_/` 看到登录界面即成功。

> 记得在阿里云安全组里**放行 8090 端口**。

### 3. 前端部署（React）

#### 创建站点

1. 宝塔 → **网站** → **添加站点**。
2. **域名**：填写你的 IP 或域名。
3. **根目录**：`/www/wwwroot/project_v2`（没有就新建）。
4. **PHP 版本**：纯静态，选"不使用 PHP"。

#### 上传前端代码

将本地 `frontend/dist` 文件夹内的**所有内容**上传到 `/www/wwwroot/project_v2`，目录结构：

```
/www/wwwroot/project_v2/
├── index.html
└── assets/
```

#### 配置 Nginx

**解决 SPA 刷新 404：**

网站 → 点击站点 → **配置文件**，在 `server { }` 块中添加：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

**（推荐）配置 /pb 反代（上 https 必做）：**

```nginx
location /pb/ {
    proxy_pass http://127.0.0.1:8090/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
```

> 配置后可将 8090 端口仅对内网开放（更安全），前端通过 `https://你的域名/pb` 访问后端。

### 4. 数据迁移（本地 → 服务器）

1. 本地 `backend/pb_data/data.db` 上传到服务器 `/www/server/pocketbase/pb_data/`，覆盖同名文件。
2. 重启后端：
   ```bash
   pkill pocketbase
   cd /www/server/pocketbase
   nohup ./pocketbase serve --http="0.0.0.0:8090" > pb.log 2>&1 &
   ```

### 5. 完成验证

- 浏览器访问 `http://<YOUR_DOMAIN_OR_IP>` 验证网页版。
- 可使用域名/IP 重新打包 Android App 分发给同事。

## 三、APK 打包

### 1. 前置准备

1. **Node.js**（已有）。
2. **Android Studio**：[下载地址](https://developer.android.com/studio)，安装后配置 SDK。
3. **Java JDK**：Android Studio 通常自带，或单独安装 JDK 17。

### 2. 构建 & 同步

```bash
cd "G:\项目管理软件_v2\frontend"
npm run build

npx cap add android   # 仅首次执行
npx cap sync          # 每次改代码后执行
```

### 3. 使用 Android Studio 打包

```bash
npx cap open android
```

1. 等待项目加载完成（首次需下载依赖，注意右下角进度条）。
2. **（可选）修改图标和名称**：替换 `app/src/main/res` 下的图标，修改 `strings.xml` 中的应用名称。
3. **测试运行**：手机开启开发者模式 + USB 调试，点击绿色 ▶️ 按钮。
4. **生成 APK**：
   - **Build** → **Generate Signed Bundle / APK** → 选择 **APK** → Next
   - 创建 Key store（记住密码），选择 Release
   - 产物位于 `android/app/release/*.apk`

### 4. 常见问题

#### 手机显示"无法连接服务器"

App 运行在手机上，需要能访问 PocketBase 地址。

**正式发布（推荐）**：通过环境变量指定 PocketBase 地址：

```powershell
setx VITE_PB_URL "http://<YOUR_PB_HOST>:8090"
```

重新打开终端后：

```bash
cd "G:\项目管理软件_v2\frontend"
npm run build
npx cap sync
```

**本地调试**：手机和电脑连同一 WiFi，将 `VITE_PB_URL` 设为电脑局域网 IP（如 `http://192.168.x.x:8090`）。

#### Android 9+ 禁止 HTTP 请求

在 `android/app/src/main/AndroidManifest.xml` 的 `<application>` 标签中添加：

```xml
android:usesCleartextTraffic="true"
```

> 建议正式环境使用 https + /pb 反代，仅内测阶段开启 cleartext。
