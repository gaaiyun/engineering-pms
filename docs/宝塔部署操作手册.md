# 宝塔面板部署操作手册 (PocketBase + React)

本手册将指导你将 `v2` 版本的工程结算管理软件部署到阿里云宝塔服务器上。

## 一、 准备工作
1. **下载 Linux 版 PocketBase**
   - 你的服务器是 Linux 系统，不能用本地的 Windows 版 exe。
   - 请下载: [pocketbase_0.22.21_linux_amd64.zip](https://github.com/pocketbase/pocketbase/releases/download/v0.22.21/pocketbase_0.22.21_linux_amd64.zip)
   - 下载后解压，你会得到一个没有后缀名的 `pocketbase` 文件。

2. **准备前端文件（重要：避免“无法登录”）**
   - 你的访问域名示例：`http://<YOUR_DOMAIN>/login`
   - 前端已支持 **PocketBase 地址自动推导**（实现见 `v2/frontend/src/lib/pocketbase.ts`）：
     - http 站点默认连接：`http://当前域名:8090`
     - https 站点默认连接：`https://当前域名/pb`（需要 Nginx 反代，避免 Mixed Content）
   - 你也可以通过 `VITE_PB_URL` 显式指定（适合 App 打包/多环境）。
   - 所以你不需要再手动改 `pocketbase.ts`，只要重新构建一次前端即可：
     ```bash
     cd v2/frontend
     npm run build
     ```
   - 如果未来你要使用 `https`，建议使用下文的 **/pb 反代** 方案（最稳）。

---

## 二、 后端部署 (PocketBase)

### 1. 上传文件
1. 登录宝塔面板 -> **文件**。
2. 进入 `/www/server` 目录，新建一个文件夹叫 `pocketbase`。
3. 进入 `/www/server/pocketbase`。
4. 点击 **上传**，把刚才下载的 **Linux版** `pocketbase` 文件传上去。
5. **重要**：在文件列表中，右键点击 `pocketbase` -> **权限** -> 勾选 **执行** (755权限) -> 确定。

### 2. 启动服务
1. 宝塔面板 -> **终端** (输入服务器密码登录)。
2. 输入以下命令启动后端（后台运行模式）：

```bash
cd /www/server/pocketbase
nohup ./pocketbase serve --http="0.0.0.0:8090" > pb.log 2>&1 &
```

3. **验证启动**:
   - 在浏览器访问 `http://<YOUR_PB_HOST>:8090/_/`
   - 如果能看到登录界面，说明后端部署成功！
   - **注意**: 记得在阿里云后台的安全组里**放行 8090 端口**，否则外网打不开。

---

## 三、 前端部署 (React)

### 1. 创建站点
1. 宝塔面板 -> **网站** -> **添加站点**。
2. **域名**: 填写你的 IP 或域名（例如 `<YOUR_DOMAIN_OR_IP>`）。
3. **根目录**: 选择 `/www/wwwroot/project_v2` (没有就新建一个)。
4. **PHP版本**: 纯静态，选择“不使用PHP”即可。

### 2. 上传前端代码
1. 宝塔面板 -> **文件** -> 进入 `/www/wwwroot/project_v2`。
2. 把本地 `v2/frontend/dist` 文件夹里的**所有内容** (index.html, assets文件夹等) 拖进去上传。
3. 现在的目录结构应该是:
   - `/www/wwwroot/project_v2/index.html`
   - `/www/wwwroot/project_v2/assets/...`

### 3. 配置 Nginx (解决刷新 404 问题)
React 是单页应用(SPA)，刷新页面时可能会报 404，需要配置 Nginx。

1. **网站** -> 点击刚才创建的站点 -> **配置文件**。
2. 找到 `server { ... }` 块，在里面添加以下内容 (或者找到现有的 `location /` 修改)：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```
3. 点击保存。

### 4.（推荐）配置 /pb 反代（未来上 https 必做）
如果你未来开了 SSL（https），浏览器会阻止网页去请求 `http://<YOUR_PB_HOST>:8090`（Mixed Content），导致“无法登录”。  
最稳的做法是在同域下反代 PocketBase：

```nginx
location /pb/ {
    proxy_pass http://127.0.0.1:8090/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
```

> 说明：配置后你可以把 8090 端口只对内网开放（更安全），并且前端可改用 `https://你的域名/pb` 访问后端。

## 四、 数据迁移 (从本地到服务器)

你在本地(Windows)配置好的表结构(Collections)，可以直接迁移到服务器，不用重新点一遍。

1. **本地**: 打开 `v2/backend/pb_data` 文件夹。
2. **服务器**: 找到 `/www/server/pocketbase/pb_data` 文件夹。
3. 把本地的 `data.db` 文件上传到服务器，**覆盖** 服务器上的同名文件。
4. **重启后端**:
   - 终端运行 `pkill pocketbase` 停止服务。
   - 再次运行启动命令 (参考上面 2.2 步)。

---

## 五、 完成
现在，你可以：
1. 用浏览器访问 `http://<YOUR_DOMAIN_OR_IP>` 使用你的网页版系统。
2. 用你的域名/IP 重新打包 Android App，发给同事安装。
