# Android 后台保活引导（PR 2 配套文档）

> 安装 EngineeringPMS v2.98+ 后，**国产 ROM 用户必须**按本文档操作，否则 1-30 分钟后台后会停止收消息。

---

## 为什么需要这一步？

EngineeringPMS 通过 **前台服务 + 长连接** 实现后台消息推送（**不使用 Firebase/FCM**）。在 Android 原生系统上，前台服务有"持久通知"保护，OS 几乎不会杀。但**国产 ROM**（MIUI / EMUI / ColorOS / OriginOS）会主动冻结/杀死后台服务，需要用户手动加白名单。

| ROM | 保活率 | 必须配置 |
|---|---|---|
| 原生 Android / Pixel | ~100% | 无需配置 |
| 三星 | ~90% | 关闭"睡眠应用"列表中的本应用 |
| 小米 / Redmi | 必须配置 | 自启动管理 + 省电策略 |
| 华为 / 荣耀 | 必须配置 | 应用启动管理 |
| OPPO / 真我 | 必须配置 | 启动管理 + 耗电管控 |
| vivo / iQOO | 必须配置 | 后台运行管理 |

---

## 通用第一步：开启"电池优化白名单"

所有 Android 6.0+ 系统都有这一选项。**第一次登录 EngineeringPMS 时会弹引导**，没引导也可以手动开启：

1. 长按 EngineeringPMS 图标 → 应用信息
2. 找到「电池」或「省电与电池」
3. 选「无限制」或「不限制」（具体文案因 ROM 而异）

---

## 小米 / Redmi (MIUI / HyperOS)

1. **自启动管理**：设置 → 应用设置 → 应用管理 → EngineeringPMS → 自启动 → **开启**
2. **省电策略**：同上界面 → 省电策略 → **无限制**
3. **锁屏后清理**：设置 → 通知中心和状态栏 → 锁屏通知 → 允许（如有此项）

**MIUI 14+ 额外**：设置 → 电池 → 应用电量管理 → EngineeringPMS → 后台无限制

---

## 华为 / 荣耀 (EMUI / HarmonyOS)

1. **应用启动管理**：设置 → 应用 → 应用启动管理 → EngineeringPMS → **关闭"自动管理"** → 手动开启 ① 允许自启动 ② 允许关联启动 ③ 允许后台活动
2. **忽略电池优化**：设置 → 电池 → 更多电池设置 → 应用启动管理（如有）

**HarmonyOS 4+ 额外**：设置 → 电池 → 应用耗电管理 → EngineeringPMS → **允许后台高耗电**

---

## OPPO / 真我 (ColorOS / realme UI)

1. **耗电管理**：设置 → 电池 → 应用耗电管理 → EngineeringPMS → **允许后台运行** + **允许后台高耗电**
2. **启动管理**：设置 → 应用管理 → EngineeringPMS → 耗电详情 → **允许后台活动**

**ColorOS 13+ 额外**：在"自启动"开关里也要打开

---

## vivo / iQOO (OriginOS / Funtouch OS)

1. **后台运行**：设置 → 电池 → 后台耗电管理 → EngineeringPMS → **允许后台高耗电**
2. **自启动**：设置 → 应用 → 权限管理 → 自启动 → EngineeringPMS → **开启**
3. **i 管家**：i 管家 App → 应用管理 → 启动管理 → EngineeringPMS → 允许自启动 + 关联启动

---

## 三星 (One UI)

1. 设置 → 应用 → EngineeringPMS → 电池 → 不受限制
2. 设置 → 应用 → 长按 EngineeringPMS → 移出「未使用应用」列表（如有）

---

## 一加 (OxygenOS)

OxygenOS 12+ 已采用 ColorOS 内核，参考 OPPO 设置。早期版本：设置 → 电池 → 电池优化 → EngineeringPMS → 不优化

---

## 魅族 (Flyme)

1. 设置 → 应用 → 应用管理 → EngineeringPMS → 后台管理 → **允许后台运行**
2. 手机管家 → 权限管理 → 自启动 → EngineeringPMS → 开启

---

## 验证是否生效

**操作步骤**：
1. 登录 app 后回到桌面
2. 确认通知栏有"工程结算管理 · 消息接收中（点击打开应用）"持久通知
3. 锁屏放置 30 分钟
4. 让另一账号给你创建任务 / 发评论
5. 屏幕亮起，应在 30 秒内收到通知

**如果没收到**：
- 检查持久通知是否还在（如不在 → 服务被杀，回到上述设置加更多白名单项）
- 设置页有「通知诊断」按钮（PR 6 之后会加），可一键检测各项状态

---

## 常见误区

❌ **"我已经允许了通知权限就够了"**  
通知权限只控制系统能否弹通知；杀进程是另一个独立机制（电池优化 / 自启动）。

❌ **"我装了 Battery Doctor / 360 安全卫士"**  
第三方电池管理类 App 会进一步杀后台。建议**关闭它们的"应用清理"功能**或卸载。

❌ **"我电池快没电了，省点电"**  
低电量模式（< 20%）下，所有 OEM 都会强化杀后台。这是系统行为，无法绕过。建议接通电源时再验证保活。

---

## 反馈与诊断

如果按本文档操作后仍无法收到后台通知，请提交以下信息给开发者：

1. 手机品牌 + 型号 + ROM 版本（设置 → 关于手机）
2. 设置中 EngineeringPMS 的所有白名单状态截图
3. 持久通知是否还在
4. 应用版本号（设置页底部）

我们会在每个新 ROM 版本发布后更新本文档。
