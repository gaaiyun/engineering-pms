## 二、手动添加测试数据

### 2.1 通过 PocketBase 管理后台

打开 `http://127.0.0.1:8090/_/` → 登录管理员账号 → 进入对应集合手动添加记录。

### 2.2 通过脚本添加特定场景数据

```bash
# 添加卡点任务数据
node scripts/add_blocked_tasks.mjs

# 批量分配任务
node scripts/batch_assign_tasks.mjs
```

## 三、测试场景数据模板

以下是手动创建测试数据时可以复制的模板。

### 3.1 创建用户

```json
{
  "username": "test_employee",
  "password": "12345678",
  "passwordConfirm": "12345678",
  "name": "测试员工",
  "email": "test@example.com",
  "role": "employee",
  "department": "工程部"
}
```

### 3.2 创建项目

```json
{
  "name": "测试项目A",
  "status": "active",
  "description": "用于测试的项目",
  "manager": "<经理用户ID>",
  "members": ["<经理ID>", "<员工1ID>", "<员工2ID>"],
  "start_date": "2026-01-01",
  "deadline": "2026-06-30",
  "progress": 30
}
```

### 3.3 创建任务

```json
{
  "project": "<项目ID>",
  "stage_name": "图纸审核",
  "status": "in_progress",
  "assignees": ["<员工ID>"],
  "created_by": "<经理ID>",
  "start_date": "2026-02-01",
  "deadline": "2026-02-28",
  "sequence": 1000,
  "priority": "high"
}
```

### 3.4 创建卡点任务

```json
{
  "project": "<项目ID>",
  "stage_name": "材料送检",
  "status": "blocked",
  "assignees": ["<员工ID>"],
  "blocker": {

```

### 3.5 创建通知

```json
{

  "content": "张经理 分配了任务「图纸审核」给您",
  "type": "task_assigned",
  "is_read": false,

  "link_id": "<任务ID>"
```

### 3.6 创建审计日志

```json
  "action_type": "update_task",
```

### 3.7 创建交接提案

```json
  "proposed_due_date": "2026-04-15",
```

## 六、自动备份

备份脚本：`scripts/auto_backup.mjs`

```bash
# 手动执行一次备份
node scripts/auto_backup.mjs

# Linux 定时任务（每12小时，保留60天）
crontab -e
# 添加：
0 */12 * * * cd /www/server/pocketbase && node /path/to/scripts/auto_backup.mjs >> /var/log/pb_backup.log 2>&1

# Windows 计划任务
schtasks /create /tn "PB_Backup" /tr "node G:\项目管理软件_v2\scripts\auto_backup.mjs" /sc hourly /mo 12
```
